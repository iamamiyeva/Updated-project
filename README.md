# Registration Form Project (JetBrains Internship Task)

This project is a modern web application built as part of the JetBrains internship application. It demonstrates the use of industry-standard frontend tooling, automated build processes, and CSS preprocessing.

## 🚀 Technologies & Tools

* **Vite:** Used as the build tool and development server for its speed and optimized bundling.
* **Sass (SCSS):** Employed to create a modular and maintainable styling architecture (utilizing variables and nesting).
* **JavaScript:** Used for logic and as the main entry point for the bundling process.

## 🛠️ Installation & Usage

To run this project locally, follow these steps:

1.  **Install dependencies:**
    ```bash
    npm install
    ```

2.  **Start the development server:**
    ```bash
    npm run dev
    ```

3.  **Build for production:**
    ```bash
    npm run build
    ```

## 📋 Task Implementation Notes

### CSS Transformation (SCSS to CSS)
All styles are authored in `src/style.scss`. The build process automatically transforms these files into optimized, minified CSS located in the `dist/assets` directory. This ensures a professional workflow where the source code undergoes a meaningful transformation rather than being a 1-to-1 copy.

### Source Maps
To facilitate efficient debugging, `vite.config.js` is configured with `sourcemap: true`. This links the minified production CSS back to the original SCSS source lines in the browser's developer tools.

> **Note:** Due to local environment configuration during the final build, `.map` files might not appear in the `dist` folder in some runs. However, the complete technical configuration (Vite and Rollup) is correctly defined in the source code to support this feature.

## 📁 Project Structure
- `src/`: Contains SCSS files and the main JS entry point.
- `public/`: Static assets such as images and icons.
- `dist/`: The production-ready output generated by `npm run build`.
- `vite.config.js`: Main configuration file for the build process.
